# 饮水记录与健康提醒应用 - 部署说明

## 1. 项目概述

本项目是一个前端单页应用，用于帮助用户记录和管理日常饮水习惯。应用采用HTML、CSS和JavaScript开发，无需后端服务器，所有数据存储在浏览器本地。

### 1.1 项目结构

```
WaterReminder/
├── index.html          # 主页面
├── css/
│   └── style.css      # 样式文件
└── js/
    └── script.js      # 核心功能脚本
```

### 1.2 技术栈

- **HTML5**：页面结构
- **CSS3**：样式设计和动画效果
- **JavaScript (ES6+)**：核心业务逻辑
- **本地存储 (localStorage)**：数据持久化

## 2. 部署环境要求

### 2.1 浏览器要求

应用支持主流现代浏览器：

- Chrome 60+
- Firefox 55+
- Safari 12+
- Edge 79+

### 2.2 设备要求

- **桌面设备**：Windows、macOS、Linux 等操作系统
- **移动设备**：iOS、Android 等操作系统
- **屏幕分辨率**：支持响应式设计，适配各种屏幕尺寸

### 2.3 网络要求

- 应用首次加载需要网络连接（加载页面资源）
- 应用运行时无需网络连接（数据存储在本地）

## 3. 部署方式

### 3.1 方式一：本地直接打开

**适用场景**：个人使用，无需公开访问

**部署步骤**：

1. 下载或克隆项目到本地
2. 找到 `WaterReminder` 文件夹
3. 双击 `index.html` 文件
4. 应用将在默认浏览器中打开

**优势**：
- 无需安装任何服务器软件
- 简单快捷，适合个人使用

**局限性**：
- 仅能在本地访问
- 部分浏览器可能限制本地文件的某些功能

### 3.2 方式二：本地HTTP服务器

**适用场景**：开发测试，本地多人访问

**部署步骤**：

#### 3.2.1 使用Python内置服务器（推荐）

1. 确保已安装Python 3.x
2. 打开命令行终端
3. 进入项目目录：
   ```bash
   cd WaterReminder
   ```
4. 启动HTTP服务器：
   ```bash
   # Python 3
   python -m http.server 8000
   
   # 或使用指定端口
   python -m http.server 9000
   ```
5. 在浏览器中访问：`http://localhost:8000` 或 `http://本地IP:8000`

#### 3.2.2 使用Node.js HTTP-server

1. 确保已安装Node.js
2. 全局安装http-server：
   ```bash
   npm install -g http-server
   ```
3. 进入项目目录：
   ```bash
   cd WaterReminder
   ```
4. 启动HTTP服务器：
   ```bash
   http-server -p 8000
   ```
5. 在浏览器中访问：`http://localhost:8000` 或 `http://本地IP:8000`

**优势**：
- 支持本地网络内多设备访问
- 避免浏览器对本地文件的限制
- 适合开发测试和小范围使用

### 3.3 方式三：静态网站托管服务

**适用场景**：公开访问，跨设备使用

**部署步骤**：

#### 3.3.1 使用GitHub Pages

1. 创建GitHub仓库
2. 将项目上传到仓库
3. 进入仓库设置
4. 找到「GitHub Pages」选项
5. 选择「main」分支，点击「Save」
6. 等待部署完成，获取访问URL

#### 3.3.2 使用Vercel

1. 注册Vercel账号
2. 连接GitHub仓库
3. 选择项目仓库
4. 点击「Deploy」部署项目
5. 获取访问URL

#### 3.3.3 使用Netlify

1. 注册Netlify账号
2. 连接GitHub仓库
3. 选择项目仓库
4. 点击「Deploy site」部署项目
5. 获取访问URL

**优势**：
- 支持全球访问
- 自动部署，无需手动维护
- 提供HTTPS支持

## 4. 配置说明

### 4.1 应用配置

应用的主要配置在 `js/script.js` 文件中，可根据需要修改：

```javascript
// 配置常量
const STORAGE_KEY = "waterPro2025";        // 存储键名
let DAILY_GOAL = 2000;                    // 默认每日目标
const MIN_FOR_STREAK = 1000;              // 连续打卡最小饮水量
const REMIND_INTERVAL = 45 * 60 * 1000;   // 提醒间隔（45分钟）
```

### 4.2 健康提醒内容

可在 `js/script.js` 文件中添加或修改健康提醒内容：

```javascript
// 健康提醒内容库
const HEALTH_TIPS = [
    "早晨空腹喝一杯温水，能唤醒身体、促进代谢",
    "饭前30分钟喝水有助于控制食量，助你减肥",
    // 可在此添加更多健康提醒
];
```

### 4.3 样式定制

可在 `css/style.css` 文件中修改应用样式，包括：

- 颜色主题
- 字体大小
- 布局结构
- 动画效果

## 6. 常见问题及解决方案

### 6.1 问题：应用无法正常加载

**解决方案**：
- 检查浏览器版本是否符合要求
- 清除浏览器缓存后重新加载
- 尝试使用其他浏览器打开

### 6.2 问题：数据无法保存

**解决方案**：
- 检查浏览器是否允许本地存储
- 清除浏览器缓存和数据后重新尝试
- 检查是否达到浏览器本地存储容量限制

### 6.3 问题：定时提醒不生效

**解决方案**：
- 确保浏览器没有被关闭或处于休眠状态
- 检查浏览器通知设置是否允许
- 部分浏览器可能会限制后台定时器，需保持浏览器标签页活跃

### 6.4 问题：应用在移动设备上显示异常

**解决方案**：
- 检查设备浏览器版本
- 确保设备屏幕分辨率支持
- 清除浏览器缓存后重新加载

## 7. 维护说明

### 7.1 数据备份

由于数据存储在浏览器本地，建议定期备份数据：

1. 在浏览器中打开应用
2. 打开开发者工具（F12 或 Ctrl+Shift+I）
3. 切换到「Application」（Chrome）或「Storage」（Firefox）标签
4. 找到「Local Storage」
5. 导出 `waterPro2025` 和 `dailyGoal` 数据

### 7.2 数据恢复

1. 在新浏览器或设备中打开应用
2. 打开开发者工具
3. 切换到「Application」或「Storage」标签
4. 找到「Local Storage」
5. 导入之前备份的数据

### 7.3 更新应用

1. 下载最新版本的项目文件
2. 替换原有文件（保留数据无需操作，数据存储在浏览器本地）
3. 刷新浏览器页面

## 8. 注意事项

1. **数据安全**：所有数据存储在用户本地，不涉及服务器传输，安全性较高
2. **隐私保护**：应用不会收集或共享任何用户数据
3. **浏览器限制**：不同浏览器对本地存储和定时器的限制可能不同
4. **数据丢失风险**：清除浏览器数据会导致应用数据丢失，建议定期备份
5. **跨设备同步**：当前版本不支持跨设备数据同步，数据仅存储在当前设备浏览器中
